Question 1 of 10

What will be the output of the following code?

function outer() { var x = 10; function inner() { console.log(x); } return inner; } const fn = outer(); fn();

Explanation:

This demonstrates a closure. The inner function "remembers" the value of x from its outer scope, even after the outer function has returned.

Question 2 of 10

Which function type is NOT hoisted in JavaScript?

Explanation:

Only function declarations are hoisted. Function expressions and arrow functions are not hoisted and must be defined before use.

Question 3 of 10

What will be the output of the following code?

const numbers = [1, 2, 3, 4, 5]; const result = numbers.map(x => x * 2).filter(x => x > 5); console.log(result);

Explanation:

First, map doubles each number: [2, 4, 6, 8, 10]. Then filter keeps only numbers > 5: [6, 8, 10].

Question 4 of 10

What is the result of the following curried function call?

const add = a => b => c => a + b + c; const result = add(1)(2)(3);

Explanation:

Currying allows partial application. add(1) returns a function that takes b, add(1)(2) returns a function that takes c, and add(1)(2)(3) returns 1 + 2 + 3 = 6.

Question 5 of 10

What will be the output of the following code?

var name = "global"; const obj = { name: "object", greet: function() { return this.name; }, greetArrow: () => { return this.name; } }; console.log(obj.greet()); console.log(obj.greetArrow());

Explanation:

Regular functions have their own 'this' binding (object), while arrow functions inherit 'this' from their lexical scope (global).

Question 6 of 10

What will be the output of the following code?

function createCounter() { let count = 0; return { increment: () => ++count, decrement: () => --count, getValue: () => count }; } const counter1 = createCounter(); const counter2 = createCounter(); console.log(counter1.increment()); console.log(counter1.increment()); console.log(counter2.increment());

Explanation:

Each call to createCounter() creates a new closure with its own count variable. counter1 and counter2 are independent.

Question 7 of 10

What will be the output of the following code?

const compose = (f, g) => x => f(g(x)); const addOne = x => x + 1; const multiplyByTwo = x => x * 2; const result = compose(addOne, multiplyByTwo)(5);

Explanation:

Composition executes from right to left: multiplyByTwo(5) = 10, then addOne(10) = 11.

Question 8 of 10

What will be the output of the following code?

for (var i = 0; i < 3; i++) { setTimeout(() => console.log(i), 100); }

Explanation:

Due to var's function scope and closure behavior, all setTimeout callbacks reference the same variable i, which has the value 3 when they execute.

Question 9 of 10

What will be the output of the following code?

function example(a, b = 2, c = 3) { return a + b + c; } console.log(example(1, undefined, 5));

Explanation:

When undefined is passed as an argument, the default value is used. So b = 2 (default), c = 5 (explicit), result = 1 + 2 + 5 = 8.
