Module Systems

Different approaches to organizing and loading JavaScript modules

ES Modules (ESM)

// Modern standard
import { utils } from './utils.js';
import React from 'react';

export const Component = () => {
  return <div>Hello World</div>;
};
Modern Standard

CommonJS

// Node.js style
const utils = require('./utils');
const React = require('react');

module.exports = {
  Component: () => {
    return React.createElement('div', null, 'Hello World');
  }
};
Legacy Node.js

AMD (Asynchronous Module Definition)

// RequireJS style
define(['./utils', 'react'], function(utils, React) {
  return {
    Component: function() {
      return React.createElement('div', null, 'Hello World');
    }
  };
});
Deprecated Legacy

Build Tools Evolution

The progression of build tools from simple to sophisticated

2010s - Grunt & Gulp

Task runners that automated repetitive tasks like minification, compilation, and testing.

// Gruntfile.js
module.exports = function(grunt) {
  grunt.initConfig({
    uglify: {
      my_target: {
        files: {
          'dest/output.min.js': ['src/input.js']
        }
      }
    }
  });
};

2015 - Webpack

Module bundler that became the standard for complex applications with dependency graphs.

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      { test: /\.js$/, use: 'babel-loader' }
    ]
  }
};

2018 - Rollup

Optimized for library development with tree-shaking and ES modules.

// rollup.config.js
export default {
  input: 'src/index.js',
  output: {
    file: 'dist/bundle.js',
    format: 'es'
  },
  plugins: [resolve(), commonjs()]
};

2020 - Vite

Next-generation build tool with instant server start and optimized builds.

// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  build: {
    outDir: 'dist'
  }
})

2021 - esbuild & SWC

Ultra-fast build tools written in Go and Rust respectively.

// esbuild
esbuild src/index.js --bundle --outfile=dist/bundle.js

// SWC
{
  "jsc": {
    "parser": {
      "syntax": "ecmascript",
      "jsx": true
    }
  }
}

Development vs Production Builds

Key differences between development and production build configurations

Development Build

  • ‚úì Fast compilation and hot reloading
  • ‚úì Source maps for debugging
  • ‚úì Detailed error messages
  • ‚úì Unminified code for readability
  • ‚úì Development server with HMR

Production Build

  • ‚úì Minified and optimized code
  • ‚úì Tree shaking and dead code elimination
  • ‚úì Asset optimization and compression
  • ‚úì Code splitting and lazy loading
  • ‚úì Performance monitoring and analytics
