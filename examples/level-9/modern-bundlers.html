Level 9: Modern Bundlers & Build Tools

Modern Bundler Landscape

Comparing the most popular build tools and their strengths

Webpack

The most mature and feature-rich bundler with extensive plugin ecosystem.

// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        use: 'babel-loader'
      },
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};
Mature Feature-rich Complex

Vite

Next-generation build tool with instant server start and optimized builds.

// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  build: {
    outDir: 'dist',
    sourcemap: true
  },
  server: {
    port: 3000,
    open: true
  }
})
Fast Modern Simple

Rollup

Optimized for library development with excellent tree-shaking.

// rollup.config.js
import resolve from '@rollup/plugin-node-resolve'
import commonjs from '@rollup/plugin-commonjs'

export default {
  input: 'src/index.js',
  output: {
    file: 'dist/bundle.js',
    format: 'es',
    sourcemap: true
  },
  plugins: [resolve(), commonjs()]
}
Tree-shaking Library-focused Optimized

esbuild

Ultra-fast bundler written in Go with minimal configuration.

// esbuild.config.js
import { build } from 'esbuild'

build({
  entryPoints: ['src/index.js'],
  bundle: true,
  outfile: 'dist/bundle.js',
  minify: true,
  sourcemap: true,
  target: 'es2020'
})
Ultra-fast Go Limited

SWC

Rust-based compiler with Babel-compatible API and extreme performance.

// .swcrc
{
  "jsc": {
    "parser": {
      "syntax": "ecmascript",
      "jsx": true
    },
    "transform": {
      "react": {
        "runtime": "automatic"
      }
    }
  }
}
Rust Fast Babel-compatible

Parcel

Zero-configuration bundler with automatic asset optimization.

// No configuration needed!
// Just run: parcel src/index.html

// Optional: .parcelrc
{
  "extends": "@parcel/config-default",
  "transformers": {
    "*.{js,jsx}": ["@parcel/transformer-js"]
  }
}
Zero-config Automatic Simple

When to Use Each Bundler

Choosing the right tool for your project needs

Webpack

  • ✓ Complex applications with many dependencies
  • ✓ Need extensive plugin ecosystem
  • ✓ Legacy codebases with specific requirements
  • ✓ Team familiar with Webpack

Vite

  • ✓ New projects with modern frameworks
  • ✓ Fast development experience priority
  • ✓ ES modules and modern JavaScript
  • ✓ React, Vue, or Svelte applications

Rollup

  • ✓ Library or package development
  • ✓ Need excellent tree-shaking
  • ✓ Multiple output formats required
  • ✓ Small bundle size critical

esbuild

  • ✓ Ultra-fast build times required
  • ✓ Simple JavaScript/TypeScript projects
  • ✓ CI/CD pipelines with time constraints
  • ✓ Minimal configuration preferred

SWC

  • ✓ Replace Babel for faster compilation
  • ✓ Large codebases with slow builds
  • ✓ React applications with JSX
  • ✓ Need Babel-compatible API

Parcel

  • ✓ Quick prototyping and demos
  • ✓ Zero-configuration setup needed
  • ✓ Small to medium projects
  • ✓ Learning and experimentation

Configuration Examples

Real-world configuration examples for different scenarios

React + TypeScript + Vite

// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  build: {
    outDir: 'dist',
    sourcemap: true,
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          utils: ['lodash', 'date-fns']
        }
      }
    }
  },
  server: {
    port: 3000,
    open: true
  }
})

Library + Rollup

// rollup.config.js
import resolve from '@rollup/plugin-node-resolve'
import commonjs from '@rollup/plugin-commonjs'
import typescript from '@rollup/plugin-typescript'

export default {
  input: 'src/index.ts',
  output: [
    {
      file: 'dist/index.esm.js',
      format: 'es'
    },
    {
      file: 'dist/index.cjs.js',
      format: 'cjs'
    },
    {
      file: 'dist/index.umd.js',
      format: 'umd',
      name: 'MyLibrary'
    }
  ],
  plugins: [resolve(), commonjs(), typescript()]
}

esbuild + Node.js

// build.js
import { build } from 'esbuild'

const isProduction = process.env.NODE_ENV === 'production'

build({
  entryPoints: ['src/index.js'],
  bundle: true,
  outfile: 'dist/bundle.js',
  minify: isProduction,
  sourcemap: !isProduction,
  target: 'node14',
  platform: 'node',
  external: ['fs', 'path']
})
