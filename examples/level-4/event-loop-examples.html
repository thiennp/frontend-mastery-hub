2. Call Stack

The Call Stack is a LIFO (Last In, First Out) data structure that stores function calls and execution contexts.

// Call Stack example function first() { console.log('First function'); second(); console.log('First function - after second()'); } function second() { console.log('Second function'); third(); console.log('Second function - after third()'); } function third() { console.log('Third function'); } first(); // Call Stack execution: // 1. first() is pushed to stack // 2. second() is pushed to stack (first() is paused) // 3. third() is pushed to stack (second() is paused) // 4. third() completes and is popped // 5. second() resumes and is popped // 6. first() resumes and is popped

3. Microtask Queue

Microtasks have higher priority than macrotasks and are executed after each macrotask completes.

// Microtask Queue examples console.log('1. Start'); setTimeout(() => console.log('2. setTimeout (macrotask)'), 0); Promise.resolve().then(() => console.log('3. Promise.then (microtask)')); queueMicrotask(() => console.log('4. queueMicrotask (microtask)')); Promise.resolve().then(() => console.log('5. Another Promise.then (microtask)')); console.log('6. End'); // Execution order: // 1. Start // 6. End // 3. Promise.then (microtask) // 4. queueMicrotask (microtask) // 5. Another Promise.then (microtask) // 2. setTimeout (macrotask)

4. Macrotask Queue

Macrotasks include setTimeout, setInterval, I/O operations, and other asynchronous operations.

// Macrotask Queue examples console.log('1. Start'); setTimeout(() => console.log('2. setTimeout 0ms'), 0); setTimeout(() => console.log('3. setTimeout 100ms'), 100); setTimeout(() => console.log('4. setTimeout 0ms (second)'), 0); setInterval(() => { console.log('5. setInterval'); clearInterval(intervalId); }, 50); const intervalId = setInterval(() => {}, 50); // Note: setInterval creates a recurring macrotask // clearInterval stops the recurring execution

5. Execution Order

Understanding the precise order of execution in the Event Loop.

Event Loop Execution Steps:

1 Execute all synchronous code
2 Execute all microtasks (Promise.then, queueMicrotask)
3 Execute one macrotask (setTimeout, setInterval, I/O)
4 Repeat steps 2-3 until all queues are empty
// Complex execution order example console.log('1. Synchronous start'); setTimeout(() => console.log('2. setTimeout 1'), 0); setTimeout(() => console.log('3. setTimeout 2'), 0); Promise.resolve().then(() => { console.log('4. Promise 1'); return Promise.resolve(); }).then(() => { console.log('5. Promise 2'); }); queueMicrotask(() => console.log('6. queueMicrotask 1')); Promise.resolve().then(() => console.log('7. Promise 3')); setTimeout(() => console.log('8. setTimeout 3'), 0); console.log('9. Synchronous end'); // Expected output: // 1. Synchronous start // 9. Synchronous end // 4. Promise 1 // 5. Promise 2 // 6. queueMicrotask 1 // 7. Promise 3 // 2. setTimeout 1 // 3. setTimeout 2 // 8. setTimeout 3

6. Blocking vs Non-blocking Operations

Understanding how blocking operations affect the Event Loop.

Blocking Operations

  • Synchronous I/O operations
  • Heavy computations
  • Infinite loops
  • Blocking DOM operations
Warning: Blocking operations freeze the UI and prevent other tasks from executing.

Non-blocking Operations

  • Asynchronous I/O operations
  • setTimeout/setInterval
  • Promises and async/await
  • Event listeners
Tip: Non-blocking operations allow the Event Loop to continue processing other tasks.
// Blocking vs Non-blocking example console.log('1. Start'); // Blocking operation (bad) function blockingOperation() { const start = Date.now(); while (Date.now() - start < 2000) { // Blocking for 2 seconds } console.log('2. Blocking operation completed'); } // Non-blocking operation (good) function nonBlockingOperation() { return new Promise(resolve => { setTimeout(() => { console.log('3. Non-blocking operation completed'); resolve(); }, 2000); }); } // This will block the Event Loop // blockingOperation(); // This won't block the Event Loop nonBlockingOperation().then(() => { console.log('4. Non-blocking operation finished'); }); console.log('5. End');

7. Performance Considerations

Optimizing code for better Event Loop performance.

// Performance optimization examples // Bad: Blocking the Event Loop function badPerformance() { const data = []; for (let i = 0; i < 1000000; i++) { data.push(i * i); } return data; } // Good: Non-blocking with chunks function goodPerformance() { return new Promise(resolve => { const data = []; let i = 0; function processChunk() { const chunkSize = 10000; const end = Math.min(i + chunkSize, 1000000); while (i < end) { data.push(i * i); i++; } if (i < 1000000) { // Use setTimeout to yield control back to Event Loop setTimeout(processChunk, 0); } else { resolve(data); } } processChunk(); }); } // Using Web Workers for heavy computations function useWebWorker() { return new Promise((resolve, reject) => { const worker = new Worker('worker.js'); worker.postMessage({ type: 'heavyComputation', data: 1000000 }); worker.onmessage = (event) => { resolve(event.data); worker.terminate(); }; worker.onerror = (error) => { reject(error); worker.terminate(); }; }); }

8. Real-world Example: Async Data Loading

A practical example of Event Loop concepts in a real application.

// Real-world example: Async data loading class DataLoader { constructor() { this.cache = new Map(); this.loadingPromises = new Map(); } async loadData(key) { // Check cache first if (this.cache.has(key)) { console.log(`Cache hit for ${key}`); return this.cache.get(key); } // Check if already loading if (this.loadingPromises.has(key)) { console.log(`Already loading ${key}, waiting...`); return this.loadingPromises.get(key); } // Start loading console.log(`Loading ${key}...`); const promise = this.fetchData(key); this.loadingPromises.set(key, promise); try { const data = await promise; this.cache.set(key, data); this.loadingPromises.delete(key); return data; } catch (error) { this.loadingPromises.delete(key); throw error; } } async fetchData(key) { // Simulate API call await new Promise(resolve => setTimeout(resolve, 1000 + Math.random() * 1000)); if (Math.random() > 0.1) { return { key, data: `Data for ${key}`, timestamp: Date.now() }; } else { throw new Error(`Failed to load ${key}`); } } // Batch loading with proper Event Loop handling async loadMultiple(keys) { console.log('Starting batch load...'); const promises = keys.map(key => this.loadData(key)); const results = await Promise.allSettled(promises); const successful = results .filter(result => result.status === 'fulfilled') .map(result => result.value); const failed = results .filter(result => result.status === 'rejected') .map(result => result.reason); console.log('Batch load completed'); return { successful, failed }; } } // Usage const dataLoader = new DataLoader(); // Load data asynchronously dataLoader.loadData('user-1') .then(data => console.log('User data:', data)) .catch(error => console.error('Error:', error)); // Load multiple data items dataLoader.loadMultiple(['user-1', 'user-2', 'user-3']) .then(results => console.log('Batch results:', results));

9. Common Pitfalls

Common mistakes and how to avoid them when working with the Event Loop.

❌ Common Mistakes

  • Blocking the Event Loop with heavy computations
  • Creating infinite loops in microtasks
  • Not handling promise rejections
  • Mixing blocking and non-blocking operations
  • Not understanding execution order

✅ Best Practices

  • Use setTimeout for yielding control
  • Break up heavy computations into chunks
  • Use Web Workers for CPU-intensive tasks
  • Handle all promise rejections
  • Understand the Event Loop execution order
// Common pitfalls and solutions // ❌ Bad: Blocking the Event Loop function badExample() { const data = []; for (let i = 0; i < 1000000; i++) { data.push(i * i); } return data; } // ✅ Good: Non-blocking with chunks function goodExample() { return new Promise(resolve => { const data = []; let i = 0; function processChunk() { const chunkSize = 10000; const end = Math.min(i + chunkSize, 1000000); while (i < end) { data.push(i * i); i++; } if (i < 1000000) { setTimeout(processChunk, 0); } else { resolve(data); } } processChunk(); }); } // ❌ Bad: Infinite microtask loop function badMicrotaskLoop() { Promise.resolve().then(() => { console.log('Microtask'); badMicrotaskLoop(); // This will block the Event Loop! }); } // ✅ Good: Controlled microtask execution function goodMicrotaskLoop(count = 0) { if (count < 10) { Promise.resolve().then(() => { console.log('Microtask', count); goodMicrotaskLoop(count + 1); }); } }
